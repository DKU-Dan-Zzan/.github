# DKU-Dan-Spot GitHub Rules

이 레포지토리는 DKU-Dan-Spot 조직의 공통 협업 규칙과 GitHub 운영 기준을 관리하기 위한 공간입니다.

참고: 이 레포는 서비스 코드 개발용이 아닙니다.  
FE/BE/AI 레포에서 공통으로 적용되는 규칙과 템플릿만 포함합니다.

---

## 목적

- 팀 전체가 동일하게 사용하는 커밋 컨벤션 정의
- Pull Request 작성 및 리뷰를 위한 공통 규칙 제공
- 모든 레포에 공통 적용되는 PR 템플릿 관리
- 협업 과정에서의 혼선 최소화

---

## 포함된 파일

| 파일 | 설명 |
|---|---|
| `CONTRIBUTING.md` | 공통 커밋 컨벤션, 브랜치 전략, PR 규칙 |
| `pull_request_template.md` | 공통 PR 템플릿 |
| `README.md` | 이 레포 안내 문서 |

---

## 실제 개발 레포

실제 개발은 아래 레포에서 진행됩니다.

- Frontend: `DanSpot_FE`
- Backend: `DanSpot_BE`
- AI: `DanSpot_AI`

각 레포에는 해당 스택에 맞는 코드 컨벤션과 설정 파일이 별도로 존재합니다.

## 운영 원칙

이 레포는 공통 규칙의 기준점(Single Source of Truth)입니다.  
규칙 변경이 필요할 경우, 본 레포에서 논의 및 수정 후 전체 레포에 반영합니다.
